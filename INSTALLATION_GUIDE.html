Installation and Configuration Guide - The Quiet Cup Café

This document provides step-by-step instructions for setting up, configuring, and deploying "The Quiet Cup Café" web application.

1. Prerequisites

Before you begin, ensure you have the following installed and set up:

  • Node.js: Download and install from nodejs.org.
  • Git: Installed on your computer for version control.
  • GitHub Account: To host your repository.
  • MongoDB Atlas Account: For cloud database hosting (cloud.mongodb.com).
  • Render Account: For free cloud hosting of your Node.js server (render.com).
  • Code Editor: VS Code (recommended).

2. MongoDB Atlas Database Setup

  1. Create a Cluster: Log in to MongoDB Atlas and create a free M0 cluster.
  2. Database User:
    • Go to Database Access under Security.
    • Click Add New Database User.
    • Set a Username (e.g., db_user) and a secure Password. Save this password.
  3. Network Access:
    • Go to Network Access under Security.
    • Click Add IP Address.
    • Choose ALLOW ACCESS FROM ANYWHERE (0.0.0.0/0).
    • Click Confirm. (This is necessary for Render's dynamic IPs to connect).
  4. Get Connection String:
    • Go back to Database.
    • Click Connect on your cluster, then choose Drivers (Node.js).
    • Copy the connection string. It will look like this:
      mongodb+srv://<username>:<password>@cluster0.example.mongodb.net/?retryWrites=true&w=majority

3. Local Development Setup

  1. Clone the Repository:

    Open your terminal and clone the project:

    git clone <your-github-repo-url>
cd "The Quiet Cup Café"
  2. Install Dependencies:

    Run the following command to install required Node.js packages:

    npm install
  3. Environment Variables:

    Create a .env file in the root folder of the project and add your MongoDB connection string:

    PORT=3000
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0...
  4. Run the Server:

    Start the application locally:

    node server.js

    Open your browser and visit http://localhost:3000.

4. Deployment to Render

  1. Push to GitHub:

    Ensure all your latest code is pushed to your GitHub repository.

    git add .
git commit -m "Ready for deployment"
git push
  2. Create Web Service:
    • Log in to your Render Dashboard.
    • Click New > Web Service.
    • Connect your GitHub account and select your repository.
  3. Configure Service:
    • Name: Choose a name for your app.
    • Build Command: npm install
    • Start Command: node server.js
  4. Set Environment Variables:
    • Scroll down to the Environment Variables section.
    • Click Add Environment Variable.
    • Key: MONGODB_URI
    • Value: Paste your exact MongoDB Connection String here.
    • Click Save Changes.
  5. Wait for Deployment:

    Go to the Events tab in Render. Wait until you see a green Deploy live message.

Security Note regarding 0.0.0.0/0:
Allowing access from anywhere (0.0.0.0/0) in MongoDB is required when hosting on free tier platforms like Render because their server IP addresses change dynamically. Your database remains secure as long as your Database Username and Password are strong and kept secret.
